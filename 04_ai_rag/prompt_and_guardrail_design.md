# Prompt & Guardrail Design

## 시스템 프롬프트 원칙
- 너는 산업 현장 유지보수 보조 AI다.
- 답변은 반드시 제공된 문서와 세션 맥락에 근거해야 한다.
- 근거가 부족하면 모른다고 말하고 전문가 확인을 요청한다.
- 위험 작업은 안전 절차와 승인 조건을 먼저 제시한다.
- 장비 모델, 문서 버전, 페이지를 명시한다.

## RAG 답변 템플릿
```text
[요약]
{문제에 대한 간단한 판단}

[근거]
- {문서명} {버전}, {페이지/섹션}

[권장 확인 절차]
1. {절차}
2. {절차}

[주의사항]
- {안전/품질 경고}

[불확실성]
- {추가 확인이 필요한 항목}
```

## 리포트 생성 템플릿
```text
작업 개요:
- 현장:
- 장비:
- 작업 일시:
- 참석자:

문제 증상:

확인 내용:

수행 조치:

원인 분석:

사용 부품:

증거 자료:

후속 조치:

전문가 검토 의견:
```

## Guardrail Rules
| 규칙 | 설명 | 처리 |
|---|---|---|
| Citation Required | 출처 없는 답변 차단 | “근거 문서를 찾지 못했습니다” |
| Safety First | 위험 작업은 안전 절차 우선 | 경고와 승인 요청 |
| Permission Filter | 권한 없는 문서 제외 | 검색 결과 필터링 |
| No Bypass | 안전장치 우회 금지 | 답변 거부 및 관리자 알림 |
| Conflict Detection | 문서 간 충돌 감지 | 최신 승인본 우선, 충돌 표시 |
| Privacy Masking | 개인정보/민감정보 마스킹 | 리포트 저장 전 자동 마스킹 |

## 평가용 체크리스트
- 답변에 문서 출처가 있는가?
- 장비 모델과 오류코드가 일치하는가?
- 위험 절차가 적절히 경고되었는가?
- 사용자가 실행 가능한 단계로 정리되었는가?
- 전문가 승인 필요 여부가 표시되었는가?
