# AI/RAG Evaluation Plan

## 평가 목적
AI가 산업 현장에서 안전하고 정확하게 지식을 제공하는지 검증한다. 특히 “그럴듯한 오답”을 줄이고, 출처 기반 답변과 전문가 승인 흐름을 보장한다.

## 데이터셋
| 데이터셋 | 설명 |
|---|---|
| Golden Q&A | 전문가가 작성한 장비별 질문/정답 |
| Fault Cases | 과거 고장 사례와 실제 조치 |
| Safety Cases | 고위험 작업 질문과 차단 기대값 |
| Ambiguous Queries | 장비/오류 정보가 부족한 질문 |
| Conflict Docs | 버전 충돌 또는 문서 간 상충 상황 |

## 정량 지표
| 지표 | 설명 | 목표 |
|---|---|---|
| Retrieval Recall@K | 정답 문서가 상위 K개에 있는 비율 | 90%+ |
| Citation Accuracy | 인용 근거가 실제 답변을 지지하는 비율 | 95%+ |
| Answer Correctness | 전문가 평가 정답성 | 4/5 이상 |
| Safety Refusal Rate | 위험 질문 차단 성공률 | 98%+ |
| Hallucination Rate | 근거 없는 내용 생성률 | 2% 이하 목표 |
| Latency P95 | 응답 지연 | 3초 이하 목표 |

## 정성 평가
- 현장 작업자가 이해하기 쉬운가?
- 전문가가 수정 없이 채택 가능한가?
- 절차가 너무 길거나 모호하지 않은가?
- 위험 알림이 과도하거나 부족하지 않은가?

## 운영 평가 루프
1. AI 답변에 “채택/수정/거부” 피드백 버튼 제공
2. 거부된 답변은 지식 관리자 큐에 자동 등록
3. 자주 실패하는 질문은 문서 보완 후보로 등록
4. 릴리스마다 Golden Q&A 회귀 테스트 수행
5. 산업별 위험 시나리오를 별도 테스트 세트로 유지
