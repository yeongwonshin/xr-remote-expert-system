# Realtime Collaboration Design

## 핵심 목표
- 현장 작업자와 원격 전문가 사이의 인지 지연을 최소화한다.
- 네트워크가 불안정해도 작업 지시가 끊기지 않게 한다.
- 주석, 음성, 영상, 체크리스트 상태가 정확히 동기화되도록 한다.

## 기술 구성
| 영역 | 설계 |
|---|---|
| 영상/음성 | WebRTC, SFU, adaptive bitrate, simulcast |
| 시그널링 | WebSocket 기반 세션 협상, ICE/TURN/STUN |
| 주석 동기화 | CRDT 또는 이벤트 소싱 기반 상태 동기화 |
| 세션 기록 | 미디어 녹화 + 이벤트 타임라인 결합 |
| 네트워크 대응 | 저대역폭 모드, 재연결, 로컬 큐, 스냅샷 모드 |

## 세션 역할
- Field Operator: 현장 영상 송출, 음성 명령, 체크리스트 수행
- Remote Expert: AR 주석, 절차 검토, 승인
- Supervisor: 세션 모니터링, 고위험 작업 승인
- AI Copilot: 매뉴얼 검색, 요약, 리포트, 위험 알림
- Guest Vendor: 제한 권한 외부 전문가

## 세션 기능
1. **Live Mode:** 실시간 영상 위 주석
2. **Freeze Mode:** 화면 고정 후 정밀 주석
3. **Evidence Mode:** 리포트용 스냅샷/영상 구간 지정
4. **Checklist Mode:** 단계별 수행 확인
5. **Low Bandwidth Mode:** 영상 품질 낮추고 음성/스냅샷 우선
6. **Offline Capture Mode:** 세션 불가 시 사진/음성/메모 저장 후 동기화

## 지연 목표
- 음성 왕복 지연: 300ms 이하 목표
- 영상 지연: 500ms 이하 목표
- 주석 동기화: 200ms 이하 목표
- AI 검색 응답: 3초 이하 목표

## 장애 대응
- 네트워크 품질 저하 시 자동으로 해상도/프레임레이트 조정
- 전문가 연결 실패 시 대기열 또는 백업 전문가로 라우팅
- 미디어 녹화 실패 시 이벤트 로그와 스냅샷 우선 보존
- AI 서비스 장애 시 수동 매뉴얼 검색으로 fallback
