# Security Architecture

## 보안 목표
- 현장 영상과 장비 정보 보호
- 외부 전문가 접근 통제
- AI/RAG 지식 접근 권한 보장
- 모든 작업 지시와 데이터 접근 감사 가능
- 고객 보안 환경에 맞는 배포 옵션 제공

## 인증/인가
| 항목 | 설계 |
|---|---|
| SSO | OIDC/SAML 연동 |
| SCIM | 사용자/그룹 자동 프로비저닝 |
| MFA | 고객 IdP 정책 준수 |
| RBAC | Technician, Expert, Supervisor, Admin, Vendor |
| ABAC | 현장, 장비, 문서등급, 시간, 네트워크 조건 반영 |
| JIT Access | 외부 벤더 1회성 접근 링크 |

## 데이터 보호
- 전송 중 암호화: TLS 1.2+ / SRTP
- 저장 시 암호화: KMS 기반 envelope encryption
- 테넌트별 암호화 키 옵션
- 영상/이미지/전사 데이터 보존기간 설정
- 민감 구역 촬영 제한과 자동 마스킹
- 다운로드 워터마크와 접근 로그

## 네트워크 보안
- Private Link/VPN 옵션
- TURN 서버 접근 제어
- 관리자 콘솔 IP allowlist
- 온프레미스 배포 시 내부 CA와 연동
- Egress 제어와 외부 LLM 호출 차단 옵션

## AI 보안
- 문서 권한 필터링 후 검색
- 출처 없는 답변 차단
- 프롬프트 인젝션 방어
- 민감정보 마스킹 후 모델 입력
- 고객별 모델/인덱스 분리
- AI 답변과 인용 출처 감사로그 저장

## 감사로그
다음 이벤트는 반드시 기록한다.
- 로그인/로그아웃
- 세션 생성/참여/종료
- 외부 사용자 초대
- 영상/리포트 다운로드
- AI 질문/답변/인용 문서
- 권한 변경
- 문서 업로드/승인/삭제
- 보존 정책 변경

## 보안 등급별 배포
| 등급 | 권장 배포 | AI 처리 |
|---|---|---|
| 일반 | SaaS | 관리형 LLM/RAG |
| 민감 | Private Cloud | 전용 인덱스/전용 모델 옵션 |
| 고보안 | On-Prem | 내부망 모델/로컬 RAG |
| 현장 제약 | Edge | 로컬 캐시/지연 동기화 |
