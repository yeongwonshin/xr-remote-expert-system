# Workflow Blueprint

## 표준 작업 흐름

```text
1. 작업 요청 생성
   ↓
2. 장비/현장/오류코드 선택
   ↓
3. AI가 관련 매뉴얼·SOP·이력 사전 검색
   ↓
4. 원격 전문가 매칭 또는 직접 초대
   ↓
5. XR 세션 시작: 영상·음성·AR 주석
   ↓
6. 작업 단계별 체크리스트 수행
   ↓
7. 증거 캡처: 사진, 영상 구간, 주석, 측정값
   ↓
8. AI 리포트 초안 생성
   ↓
9. 전문가/관리자 검토 및 승인
   ↓
10. CMMS/EAM/티켓 시스템 업데이트
```

## 긴급 장애 복구 플로우
1. 작업자가 QR/NFC/장비 ID를 스캔한다.
2. 오류코드와 증상을 음성으로 입력한다.
3. 시스템이 관련 매뉴얼, 과거 고장 사례, 안전 유의사항을 자동 로딩한다.
4. 긴급도에 따라 전문가 큐에 배정된다.
5. 전문가가 실시간 화면을 보고 AR 주석으로 조치 위치를 지정한다.
6. AI가 “현재 3단계에서 필요한 확인 항목”을 화면에 띄운다.
7. 작업 완료 후 AI가 원인·조치·증거·후속 작업을 정리한다.

## 외부 벤더 지원 플로우
1. 내부 관리자가 외부 벤더용 1회성 접근 링크 생성
2. 접속 범위: 특정 세션/특정 장비/특정 문서로 제한
3. 화면 녹화와 감사로그 자동 활성화
4. 벤더 주석과 발언은 리포트에 별도 표시
5. 세션 종료 시 권한 자동 폐기

## 리포트 생성 플로우
- 입력: 세션 메타데이터, 음성 전사, 주석, 사진, 체크리스트, AI 검색 기록
- 처리: 요약 → 원인/조치 분류 → 근거 이미지 매칭 → 누락 항목 검사
- 출력: 고객 제출용 리포트, 내부 기술 리포트, 티켓 업데이트용 JSON
